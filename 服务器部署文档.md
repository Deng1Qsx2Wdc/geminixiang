## 系统要求

### 操作系统
- Ubuntu 18.04+ / Debian 10+
- CentOS 7+ / RHEL 7+
- 其他支持 systemd 的 Linux 发行版

### 软件依赖
- Python 3.7+
- pip3
- 网络连接（用于下载依赖和访问 Gemini API）

### 硬件要求
- 最低：1 CPU 核心，512MB 内存
- 推荐：2 CPU 核心，1GB 内存

## 快速部署

### 方法一：使用自动部署脚本（推荐）

1. **上传项目文件到服务器**
   ```bash
   # 使用 scp 上传（在本地电脑执行）
   scp -r gemininixiang/ user@服务器IP:/home/user/
   
   # 或使用 git clone
   git clone https://github.com/erxiansheng/gemininixiang.git
   cd gemininixiang
   ```

2. **运行部署脚本**
   ```bash
   # 赋予执行权限
   chmod +x deploy.sh
   
   # 运行部署脚本（使用普通用户，不要用 root）
   ./deploy.sh
   ```

3. **按照提示完成部署**
   - 脚本会自动检查环境
   - 安装依赖
   - 配置防火墙
   - 创建系统服务
   - 启动服务

### 方法二：从 ZIP 文件部署

1. **上传 ZIP 文件到服务器**
   ```bash
   # 将项目打包为 zip 文件后上传
   scp gemininixiang.zip user@服务器IP:/tmp/
   ```

2. **在服务器上执行部署**
   ```bash
   # SSH 登录服务器
   ssh user@服务器IP
   
   # 解压文件
   unzip /tmp/gemininixiang.zip -d /tmp/
   
   # 进入项目目录
   cd /tmp/gemininixiang
   
   # 运行部署脚本
   chmod +x deploy.sh
   ./deploy.sh
   ```

## 手动部署

如果自动部署脚本无法使用，可以按照以下步骤手动部署：

### 1. 准备项目文件

```bash
# 创建项目目录
mkdir -p ~/gemini-api
cd ~/gemini-api

# 上传或复制项目文件到此目录
# 确保包含以下文件：
# - server.py
# - requirements.txt
# - config_data.json（可选，会自动创建）
```

### 2. 安装 Python 依赖

```bash
# 创建虚拟环境
python3 -m venv venv

# 激活虚拟环境
source venv/bin/activate

# 升级 pip
pip install --upgrade pip

# 安装项目依赖
pip install -r requirements.txt
```

### 3. 配置防火墙

**Ubuntu/Debian (UFW):**
```bash
sudo ufw allow 8000/tcp
sudo ufw reload
```

**CentOS/RHEL (Firewalld):**
```bash
sudo firewall-cmd --add-port=8000/tcp --permanent
sudo firewall-cmd --reload
```

**云服务器安全组：**
- 在云服务商控制台（阿里云、腾讯云、AWS 等）配置安全组规则
- 开放端口：8000（TCP）

### 4. 创建 Systemd 服务

```bash
# 创建服务文件
sudo nano /etc/systemd/system/gemini-api.service
```

将以下内容写入文件（**注意修改路径和用户名**）：

```ini
[Unit]
Description=Gemini OpenAI API Server
After=network.target

[Service]
Type=simple
User=你的用户名
WorkingDirectory=/home/你的用户名/gemini-api
Environment="PATH=/home/你的用户名/gemini-api/venv/bin"
ExecStart=/home/你的用户名/gemini-api/venv/bin/python /home/你的用户名/gemini-api/server.py
Restart=always
RestartSec=10
StandardOutput=journal
StandardError=journal

[Install]
WantedBy=multi-user.target
```

保存后执行：

```bash
# 重新加载 systemd
sudo systemctl daemon-reload

# 启用服务（开机自启）
sudo systemctl enable gemini-api

# 启动服务
sudo systemctl start gemini-api
```

### 5. 验证部署

```bash
# 检查服务状态
sudo systemctl status gemini-api

# 查看服务日志
sudo journalctl -u gemini-api -f

# 测试本地访问
curl http://localhost:8000/admin
```

## 配置说明

### 服务器配置（server.py）

主要配置项位于 `server.py` 文件开头：

```python
API_KEY = "sk-gemini"          # API 密钥（客户端使用）
HOST = "0.0.0.0"               # 监听地址（0.0.0.0 表示所有网络接口）
PORT = 8000                    # 监听端口（默认 8000）
ADMIN_USERNAME = "admin"       # 后台管理用户名
ADMIN_PASSWORD = "admin123"    # 后台管理密码
```

**注意：** 如果使用 `deploy.sh` 脚本，脚本中配置的防火墙端口是 8001，但 `server.py` 默认端口是 8000。请确保两者一致：
- 要么修改 `server.py` 中的 `PORT = 8001`
- 要么修改 `deploy.sh` 中的端口号为 8000

**修改配置后需要重启服务：**
```bash
sudo systemctl restart gemini-api
```

### 配置文件（config_data.json）

首次运行会自动创建 `config_data.json` 文件，包含：
- Cookie 配置（用于访问 Gemini API）
- 代理设置（可选）

**配置方式：**
1. 访问后台管理页面：`http://服务器IP:8000/admin`
2. 使用默认账号密码登录：`admin` / `admin123`
3. 在后台页面配置 Cookie 和代理

### 修改端口

如果需要修改端口（例如改为 8001）：

1. **修改 server.py**
   ```python
   PORT = 8001
   ```

2. **修改防火墙规则**
   ```bash
   # 删除旧端口
   sudo ufw delete allow 8000/tcp
   # 添加新端口
   sudo ufw allow 8001/tcp
   ```

3. **修改 systemd 服务文件**（如果服务文件中有硬编码端口）

4. **重启服务**
   ```bash
   sudo systemctl restart gemini-api
   ```

## 常见问题

### 1. 服务无法启动

**检查日志：**
```bash
sudo journalctl -u gemini-api -n 50
```

**常见原因：**
- Python 路径错误：检查虚拟环境路径是否正确
- 端口被占用：`sudo lsof -i :8000`
- 权限问题：检查文件权限和用户设置
- 依赖缺失：重新安装依赖 `pip install -r requirements.txt`

### 2. 无法从外部访问

**检查清单：**
1. **防火墙是否开放端口**
   ```bash
   sudo ufw status
   # 或
   sudo firewall-cmd --list-ports
   ```

2. **云服务器安全组**
   - 登录云服务商控制台
   - 检查安全组规则是否开放 8000 端口

3. **服务是否监听正确地址**
   ```bash
   sudo netstat -tlnp | grep 8000
   # 应该显示 0.0.0.0:8000 或 :::8000
   ```

4. **服务器 IP 地址**
   ```bash
   hostname -I
   # 或
   ip addr show
   ```

### 3. API 调用失败

**可能原因：**
- Cookie 未配置或已过期
- 网络连接问题
- 代理配置错误

**解决方法：**
1. 访问后台管理页面检查 Cookie 配置
2. 查看服务日志：`sudo journalctl -u gemini-api -f`
3. 测试网络连接：`curl https://generativelanguage.googleapis.com`

### 4. 服务频繁重启

**查看详细日志：**
```bash
sudo journalctl -u gemini-api -n 100 --no-pager
```

**常见原因：**
- 代码错误导致崩溃
- 内存不足
- 依赖包版本冲突

### 5. 忘记后台密码

**重置方法：**
1. 停止服务：`sudo systemctl stop gemini-api`
2. 编辑 `server.py`，修改 `ADMIN_PASSWORD`
3. 重启服务：`sudo systemctl start gemini-api`

## 维护与更新

### 更新项目

1. **备份当前配置**
   ```bash
   cd ~/gemini-api
   cp config_data.json config_data.json.bak
   ```

2. **停止服务**
   ```bash
   sudo systemctl stop gemini-api
   ```

3. **更新代码**
   ```bash
   # 如果使用 git
   git pull
   
   # 或重新上传新版本文件
   ```

4. **更新依赖（如有变更）**
   ```bash
   source venv/bin/activate
   pip install -r requirements.txt --upgrade
   ```

5. **启动服务**
   ```bash
   sudo systemctl start gemini-api
   ```

### 完全重新部署

如果需要完全清理并重新部署，可以使用清理脚本：

```bash
# 1. 停止并删除服务
sudo systemctl stop gemini-api
sudo systemctl disable gemini-api
sudo rm /etc/systemd/system/gemini-api.service
sudo systemctl daemon-reload

# 2. 删除项目目录
rm -rf ~/gemini-api

# 3. 按照部署步骤重新部署
```

### 备份与恢复

**备份：**
```bash
# 备份整个项目目录
tar -czf gemini-api-backup-$(date +%Y%m%d).tar.gz ~/gemini-api

# 仅备份配置文件
cp ~/gemini-api/config_data.json ~/config_data.json.backup
```

**恢复：**
```bash
# 解压备份
tar -xzf gemini-api-backup-YYYYMMDD.tar.gz -C ~/

# 恢复配置文件
cp ~/config_data.json.backup ~/gemini-api/config_data.json
```

## 附录

### 项目文件结构

```
gemini-api/
├── server.py              # 主服务文件
├── requirements.txt       # Python 依赖
├── config_data.json      # 配置文件（自动生成）
├── deploy.sh             # 自动部署脚本
├── venv/                 # Python 虚拟环境
└── ...                   # 其他项目文件
```

### 端口说明

- **8000**：HTTP API 服务默认端口（可在 server.py 中修改）
- **注意：** 如果修改了端口，需要同步更新：
  1. `server.py` 中的 `PORT` 变量
  2. 防火墙规则
  3. systemd 服务文件（如有硬编码）
  4. `deploy.sh` 脚本中的端口号（如使用脚本部署）

### 环境变量（可选）

可以通过环境变量覆盖配置：

```bash
# 在 systemd 服务文件中添加
Environment="API_KEY=your-key"
Environment="PORT=8001"
```
